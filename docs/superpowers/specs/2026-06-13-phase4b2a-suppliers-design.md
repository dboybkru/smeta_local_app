# Фаза 4b-2a — Управление поставщиками (фронтенд): дизайн

> Брейншторм-дизайн, 2026-06-13. Под-фаза 4b-2a (первая из 4b-2). Закрывает блокер: на свежем проде «Поставщик» в мастере импорта пуст и нет UI создания поставщика (пробел фазы 2b). 4b-2b (админ-UI AI-конфига) — отдельный спек.

## Цель

Дать возможность заводить поставщиков из интерфейса: отдельная страница управления + инлайн-создание прямо в мастере импорта (чтобы импорт прайса не упирался в пустой список).

## Бэкенд — без изменений

Уже есть: `GET /api/suppliers` (любой `require_active`), `POST /api/suppliers` (`require_admin`, `409` если имя занято). Этого достаточно. **Удаление не делаем** — поставщик связан с прайс-листами (`PriceList.supplier_id`) и позициями каталога; удаление осиротит данные. (Мягкое архивирование — при необходимости позже.) Авто-парс поставщика из файла прайса — отложен (отдельная логика определения).

## Фронтенд

Стек/паттерны как 3b/4b-1. Доступ: всё под admin (бэкенд `POST` уже `require_admin`; страница и ссылка — только admin, как «Импорт»/«Уровни цен»).

| Файл | Ответственность |
|---|---|
| `src/api/catalog.ts` | + `createSupplier(name)` → `POST /suppliers` |
| `src/pages/SuppliersPage.tsx` | `/admin/suppliers`: список + форма «Добавить» |
| `src/pages/ImportPage.tsx` | инлайн «+ новый поставщик» рядом с дропдауном |
| `src/App.tsx` | + маршрут `/admin/suppliers` под `RequireAuth` |
| `src/components/AppHeader.tsx` | + ссылка «Поставщики» (admin) |

### SuppliersPage (`/admin/suppliers`)
- При загрузке — `listSuppliers()`; список имён.
- Форма «Добавить»: поле имени + кнопка → `createSupplier(name)` → добавить в список, очистить поле. На `409` (ApiError.status===409) показать «Поставщик с таким именем уже существует».
- Пустой список → подсказка «Поставщиков пока нет — добавьте первого».

### ImportPage — инлайн-создание
- Рядом с `<select aria-label="Поставщик">` — кнопка «+ новый». Клик → показать инлайн-инпут + «Создать»/«Отмена».
- «Создать» → `createSupplier(name)` → добавить в `suppliers`, выставить `supplierId` на нового, скрыть инпут. На `409` — сообщение, не сбрасывать ввод.
- Существующий поток импорта (inspect→map→import) не меняется.

### AppHeader
Ссылка «Поставщики» → `/admin/suppliers`, видна только admin (`isAdmin`), рядом с «Импорт»/«Уровни цен».

## Тестирование (Vitest, мок fetch + мок useAuth, паттерн 3b/4b-1)

- `SuppliersPage`: рендер списка; создание (POST → имя появилось, поле очищено); `409` → сообщение об ошибке.
- `ImportPage`: «+ новый» → ввод имени → «Создать» → `createSupplier` вызван, новый поставщик выбран в дропдауне.
- `AppHeader`: «Поставщики» видна admin, скрыта у estimator/viewer.
- Сборка/линт чисто.

## Решения (зафиксировано)

1. Только список + создание (без удаления — осиротит данные).
2. Создание доступно из двух мест: страница `/admin/suppliers` и инлайн в мастере импорта (общий `createSupplier`).
3. Всё под admin.
4. Авто-парс поставщика из прайса — отложен.

## Отложено (не в 4b-2a)

- Админ-UI AI-конфига → 4b-2b.
- Удаление/архивирование поставщиков.
- Авто-определение поставщика из файла прайса.
